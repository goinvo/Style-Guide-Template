Report

Layout

When confining content to a 1024px width, allow legend 107px, with 15px of gutter between the legend and the chart. Allow the report title 575px width, with a 15 px gutter between the title and the report options.

15px of space between the navigation and the title / report options area, allowing for space for more complex controls and title editing. Align report options with the title such that the baseline for main text is the same. 30px of space between title / report options area and the chart area. Align the top of the chart to the top of the legend. 15px spacing between Analysis period label and any otherx-axis labelling.

Report Title

Allow user to enter edit mode by clicking on the edit icon or by clicking on the title itself. Exit edit mode by clicking OK or anywhere outside the input.

Scripts and styles below are used to demonstrate the desired look and behaviour. Alternatives are welcome as long as the appearance and behaviour are replicated.

Report Legend

Report legend. Allow for 100% height of the report container in the case that there are so many segments that the user will need to scroll. Once that point is reached, allow the user to type in the progressive search and scroll to that segment, highlighted with font-weight: bold.

Segments added after initial load are injected into the list alphabetically, but are assigned colors continuing from the end of the original list.

Selecting segments from the hidden list does not add them to the above list as these are exceptions to what's hidden.

Styles below may vary once in implementation due to rendering in svg through d3.js, but the intent is to match the sizing and spacing.
